Device Credentials

Used to push text to the C1720 / C1710 display.

Polling

The app adapts automatically: faster during live matches, slower when idle. 30 s is the hard minimum. The Manual Cap (optional) overrides to a slower rate — useful for protecting API quota.

Display Appearance

⚙ Manual Override

Inject a custom message directly onto the display for a fixed duration. The ticker resumes automatically when the override expires.

🏁 Goal Strobe

When a tracked team scores, the display flashes in the team's kit colors. Audio plays in parallel. Each flash alternates the primary kit and inverted colors.

🏆 Per-Team Overrides

Override clip ID and flash count for individual tracked teams. 0 = use global setting.

🔗 Goal Event Push (Webhook)

When a tracked team scores, the app POSTs a JSON payload to this URL. Compatible with Home Assistant, Discord bots, Zapier, IFTTT, and any HTTP endpoint.

🎼 Audio Alerts

Goal sounds are streamed to the C1720/C1710 speaker via VAPIX transmit.cgi. Volume is baked into the PCM data — the device output level is never changed. Click Upload Clips once after installing the ACAP to store the bundled MP3s on the device.

🔬 Diagnostics

Use these checks to isolate problems at each layer of the stack.

Resets the rate-limit timer and triggers an immediate fetch from the live API (or mock if Demo Mode is on). Check the Live tab after ~5 seconds.

            
            

        


        
        

            
4 — VAPIX Display

            
Sends a test message directly to the C1720/C1710 display. If nothing appears, check device credentials in Config.
